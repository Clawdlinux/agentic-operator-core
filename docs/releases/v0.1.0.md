# v0.1.0 Release Notes

Release date: 2026-03-18

## Summary

Initial open-source baseline of Agentic Operator as a Kubernetes operator for isolated, policy-aware AI agent workloads.

This release replaces a previously misnumbered `v1.0.0` baseline tag so the public open-source version history starts at `v0.1.0`.

## Highlights

- AgentWorkload CRD and reconciliation loop baseline.
- Argo Workflows DAG orchestration integration.
- Cilium-oriented policy and isolation templates.
- Shared LiteLLM proxy integration points.
- Shared Browserless pool integration points.
- Helm umbrella chart baseline for operator and shared components.

## Documentation updates included in this release

- Updated top-level README for cold-start-safe quickstart.
- Added clear open/private boundary guidance.
- Added architecture diagram in README.
- Added CONTRIBUTING guidelines.
- Added GitHub issue templates for bugs and feature requests.

## Upgrade and compatibility notes

- This is an initial baseline release for open-source adoption.
- API is v1alpha1 and subject to iterative hardening.
- Keep private licensing and billing logic in `agentic-operator-private`.

## Known limitations

- Some advanced enterprise and licensing/billing capabilities are intentionally excluded from this open repository.
- Production hardening should follow docs and environment-specific overlays.

## Validation checklist

- CRD applies cleanly: `config/crd/agentworkload_crd.yaml`
- Local chart install path works from `charts/`
- Sample AgentWorkload can be created from `config/agentworkload_example.yaml`

## Suggested tag command (run from clean main)

```bash
git checkout main
git pull --ff-only
git tag -a v0.1.0 -m "v0.1.0: Initial open-source baseline"
git push origin v0.1.0
```
